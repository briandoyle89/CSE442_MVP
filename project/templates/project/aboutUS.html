Sharing is Learning


UB_Liberator is built to help students share learning resources. You can download old homeworks, exams, code, etc. for CSE courses at UB. Furthermore, you can upload similar files to share with other students.

Project Description

From the Developers of UB_Liberator, bits_please-

Our website UB_Liberator is designed to allow students, specifically CSE students at UB, to have access to another resource for class notes and materials they may have lost or not completed correctly. Through being able to search for files, and rate them through an upvote/downvote system, students can find the information they need to succeed to the best of their capabilities. Files uploaded by users are their own works and any transgressions of academic integrity or other code of conduct is not the fault of the developers, bits_please.

Description for Users

Please follow this link to our website-
https://ubliberator3.pythonanywhere.com/login_user?next=/course_list . Upon creating an account, you will be taken to the main Dashboard page of our site, where you can begin to find the documents and pages you need to do the best that you can in your courses! The left side of the page holds the tools you need to browse through our current course list, view your previous downloads and uploads, and upload a document.

Use Cases-

1. Registering-
User - A typical user is a student who is taking CSE courses at UB who needs additional resources to help ease through his/her CSE course

Why is it related to the description - As mentioned in the ReadMe, our project is a file sharing website, and hence for a user to use this website, he/she has to register which makes it an appropriate use case.

As a UB CSE student upon opening the site, I can press the button on the left side of the screen “Don’t Have an Account? Click Here.” to be taken to a screen where a valid format of email, a username that hasn’t been taken yet, and a password that is not blank, can be entered into their corresponding boxes to the left of their texts (Email, Username, and Password) and after using the keyboard to type your responses and clicking submit, your account is created.

2. Downloading a File -
User - A typical user is a student who is taking CSE courses at UB who needs additional resources to help ease through his/her CSE course.

Why is it related to the description - As mentioned in the user story section of our ReadMe, our project is a file sharing website where a user can download study material from our website, thereby making ‘downloading a file’ an appropriate use case.

As a CSE student at UB, after logging into the site, I can view on the dashboard the classes CSE115 and CSE442, and their corresponding buttons “Choose This Class” below them. By clicking on their respective “Choose This Class” buttons, I can then see the list of materials for the course available for download. Then, by navigating down the list of file names and their respective “Download this File” buttons, I can find a file I want, and click the “Download this File” button, to begin the download. I can check the downloads folder of the computer I used to click the “Download this File” button, to see if it was successfully downloaded. This will verify that the download functionality works.

3. Uploading A File-
User - A typical user is a student who is taking CSE courses at UB who needs additional resources to help ease through his/her CSE course

Why is it related to the description - As mentioned in the user story section of our ReadMe, our project is a file sharing website where a user can upload study materials from our website for other users to leverage it, thereby making ‘uploading a file’ an appropriate use case.

As a UB CSE student, after logging into the site, and being taken to the Dashboard automatically, I can click on the button on the left navigation bar titled “Upload a DOCUMENT” and after clicking into the box to the right of “File name:” and entering information for a file name, choosing a course, and choosing a file, I can click upload to upload the file to the specified course.

4. Searching Files By Class-
User - A typical user is a student who is taking CSE courses at UB who needs additional resources to help ease through his/her CSE course

Why is it related to the description - As mentioned in the user story section of our ReadMe, our project is a file sharing website where a user can download study materials from our website for other users to leverage it. In order to search for relevant documents, we’ll be implementing a search files by class functionality for our beta-release which will be an appropriate use case.

As a UB CSE student, after logging into the site and being taken to the Dashboard, the user can select from a drop down menu button labelled Select Course, the course he/she wants the study materials for. Once user selects the course and clicks the search button, all the files associated with that course will be populated on the screen, and then the user can download any of the files that he/she wants.

Testing Procedures-
1. Uploading a File, can search it to confirm it has been uploaded.
Target-The target is to test the process of uploading a file to our site, UB_Liberator, from the user’s computer.

Testing Procedure- A user must log in/register at our UB_Liberator landing page. After this, the user is taken to the Dashboard page, where there is a blue-colored button called --”Upload a DOCUMENT” located below the “My Uploads” button at the 4th position on the left navigation bar of the page. On clicking it, the user is told to enter the file_name, and is asked to select the course for which they are uploading the file. Then there is a button called “Choose file” which allows the user to select the file to be uploaded from their local system. Next is the “Upload” button which the user clicks to upload their file to the website. After filling in the file_name, course, and have chosen the file to be uploaded, the “Upload” button must be clicked.

Expected Outcome-After following the testing procedure step by step, the file should show up in the “My Uploads” section (located at the 2nd position of the navigation bar positioned on left-most side of the page). “My Uploads” is a button above the “Upload a DOCUMENT” button, and by clicking it, a user can see the files they have uploaded to the website. Hence, verifying that the Upload functionality actually works. The user can also double check by clicking back to the dashboard, selecting the class they chose for the file, and scrolling down the page until they can view their file in that way.

2. Registering

Target-In order to enable the user story, the user has to register and make an account on our website. The target is to test the procedure for registering on to our UB_Liberator site, using their login credentials viz. Email id, username and password which the user can choose.

Testing Procedure - On the Login page click on the “Don’t Have an Account? Click Here” button to create an account. On doing so, the user will be directed to another login page which will have the following input boxes labelled - Email, Username and Password. In the email box input any email of your choosing It doesn’t specifically have to be your UB mail address). Then click on the “Password” and the “Username” box and input any username and password of your choosing. After inputting all three click the “Submit” button. Once the information is submitted you will be redirected back to the Login page. On the login page input the same username and password that you used for registering and click “Submit”. You have now created an account and logged in successfully.

Expected Outcome- After the student clicks “Submit”, the account should be fully created, and then when the page goes back to the login page, you should be able to fill out the username and password boxes by clicking on them and typing in your appropriate credentials. Then, clicking “Submit” should log you in and direct you to the main Dashboard page.
